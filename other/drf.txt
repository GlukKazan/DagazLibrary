- Команда log формирует dsg-описание хода
  - Части хода формируются по паре критериев
    - order (0 по умолчанию)
    - лексикографический порядок содержимого строки (повторяемость dsg-нотации)
  - При совпадении нотации, ходы считаются идентичными (повторения игнорируются)
- В and и or операнды выполняются по порядку перечисления, по сокращённой схеме
- is-piece? сначала проверяет текущее поле и если оно пусто - переносимую фигуру
- drop-all в конце хода и take-piece в начале формируются автоматически
- Поддержка "частичных" ходов в Шашках
  - post вызывается после каждой итерации хода repeat
  - Если тип фигуры изменился в процессе хода (Русские шашки) вызывается начало цикла repeat другой фигуры (поиск цикла осуществляется про имени)
  - Нотация используется для визуализации перемещения фигуры, но информация об удалении фигур выбирается непосредственно из State
- Поддержка коллекционных игр
  - Возможность программной генерации кода в индивидуальной конфигурации игрока, связанной с игрой
  - Криптографическая подпись индивидуальных конфигураций
- change-owner! (в ming-mang) журналируется
  - Если ход принимается (все проверки check проходят), выполняются и журналированные команды
  - Повторные команды change-owner!, для тех-же фигур, игнорируются
